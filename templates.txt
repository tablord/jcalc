

allowed structure for templates


a template must be of one single head tag having a class=ELEMENT

   <DIV class=ELEMENT template="example#1">
     any html strucure inside
   </DIV>

the ELEMENT class is intended for the EDI to know what can be the selectedElement when the user clicks on it.

inside the ELEMENT, the html structure is free.

any tag that has the class EDITABLE will get the contentEditable=true when the element is selected and contentEditable=false when not selected
please note that in IE7 <TABLE> nor <TD> <TH>.. are editable

if a tag has the class CONTAINER it will accept to insert other ELEMENT. optionally the property "templates" specifies all template names that are accepted


   <DIV class=ELEMENT itemprop="example#1" itemscope itemtype="http://tablord.com/template/example#1">
     ....
     <DIV container="emails" templates="email:email#1 email:email#2">
        ...here the EDI will accept to insert template email or address
        <DIV class=ELEMENT itemprop="email" itemscope itemtype="http://microdata.com/repository/email#1">
          <SELECT itemprop="purpose"><OPTION>office</OPTION><OPTION>home</OPTION></SELECT>
          <SPAN itemprop="emailAddress">marc9nicole@gmail.com</SPAN>
        </DIV>
        <DIV class=ELEMENT itemprop="email" itemscope itemtype="http://microdata.com/repository/email#2">
          <SELECT itemprop="purpose"><OPTION>office</OPTION><OPTION>home</OPTION></SELECT>
