2 Introduction

2.1 What is 
Tablord?






A bit of Word, a bit of wikipedia, a bit of Excel and a bit of python "notebook".


Tablord is born out a frustration: if you want to do a descent report for a scientific publication or a business plan, or any document that mixes both text and some calculation (in fact almost anything that is not poetry), you eventually come to a big dilemma: should I use Word our should I use Excel? of course Word because I need quite a lot of explanation and of course Excel because I use quite a bit of calculation. At the end, you cut and paste excel tables in Word. So far so good (in fact already some pain, but OK, worth the pain). But then the good news is someone read your work! Here is the bad news: someone read your work, find it very interesting and "just" point out: "what about if â‚¬ / $ is 10% higher?" Ouuups! of course, most probably you can change some cells in Excel and it will do the maths... and then you cut & paste again in Word, re-read all the text to see where you included some figure that now are no longer valid and and and... you fear the next question since you realize this job is very time consuming. 




 


The second frustration comes from Excel itself: as long as you do some simple calculation, this is extremely efficient. but as soon as you start to do some complex work, this becomes extremely frustrating since complex equations with meaningless cells name like A3+C5+D2/2 are almost impossible to maintain.


As well as surprisingly, all situation where you have a certain number of tables of data: it become very complicated since the presentation of data enter in conflict with the maintainability of the sheet. Excel has some good functions but most people don't know them because it becomes extremely difficult to manage.


 


I past position I faced many time this dilemma and dreamed of something better, or at least different. At that time I was impressed by Wikipedia, that has some very specific macro mechanism that helps to do some job, but could not replace Excel and python Notebook that is excellent to do complex maths, but has almost no capability to edit text. and notebook is relatively complex to install, so not for anyone.


 


Of course had no time to make this dream come true, but as life is a continuous surprise, all of a sudden I had too much time... this was the occasion to make this dream come true




2.2 Fundamentals of 
Tablord






The fundamentals are


    

  • Web based: browser based, so decently OS independent, very strong ecosystem. No fat client (in fact the fat is included in the browser) 

  • all maths are JavaScript and al least on the beginning only on the client side. JavaScript is very well supported, so no short term risk of obsolescence and even if I personally find python more elegant than JavaScript, JavaScript deserves to be considered as a very powerful language, and of course it has plenty of open source libraries to be used inside Tablord. no special language just plain Java-Script 

  • simple file based system for the beginning. In the future some work will be done in servers. 

  • A system that can mix text and calculation in a very fluid manner




2.3 What is not 
Tablord




Tablord is not Excel compatible, nor Word compatible etc... You suffer in one of this software... well you will probably continue to suffer for a while to tweak macros as long as you do not have the courage to jump to 
Tablord

2.6 Code Execution 






2.6.1 Run Until Selected button or ctrl-Enter






While editing an element, you can either click on the "run until selected" 
or press ctrl-Enter. Both will run all code element until (and including) the currently selected Element. 



Please note that every run is done in a new fresh environnement, so every previously declared variable through v, table... are deleted before execution.




2.6.2 Run All and auto run






if the sheet is marked with auto run (by default) it will be executed fully on the opening of the sheet.


you can re-execute everything by pressing the run all button 
. In both cases, the execution start by creating a fresh new variables environnement

3.1.1 Naming conventions






In 
Tablord
one should avoid starting a variable name by _ since those are reserved for the library usage (like _simulation). 



If you do so, you will be sure that this name space is for you and no upgrade of 
Tablord
will interfear in your variables. 



For the rest, it's up to you, but common good practice for JavaScript is to camelCase every word but the first and avoid underscore.


 


the library follows those common conventions:


a function stating with a CapitalCase is a constructor (~ a Class)

Unfortunally no modern language supports properly units (which is a shame). JavaScript is no exception :-( . So, even if this is far from perfect, some provision have been given in 
Tablord
while remaining pure JavaScript.

3.5 output






so far all output where just the result of the last statment displayed by the EDI in a proper format. Sometimes you want to better control the output of your code Element.


For every code Element an output object is created. this output object is an HTML object with 2 extra properties: codeElement and outputElement that points to the code DOM Element and the output DOM Element.


 


You can use output to build step by step the result of your code like in the following example

for (var i = 0; i<10;i++) {

}

3.6 function delaration






such a spreadsheet would be of no use if we can not make calculation.


 


As we already have declared 
currentPrice = 1234 USD
we can now define the following 
costOfGood = 1000 USD 

so now we can calculate the margin as a formula
v('margin$USD',f('currentPrice$USD-costOfGood$USD')).view()
margin = 234 USD





let explain a little bit of the magic of f()


f takes a string parameter that will become the body of the function
function (row,col) {
  with (jc.vars) {
    with (row || {}) {
      return // your body will be here
    }
  }
}


let puts aside for a while (row,col). as we are inside a with(jc.vars), we have direct access to all previously defined variables.


This function will be stored inside margin$USD so any attempt to get its value will first recalculate the function. 


Please note that return is automatically added at the beginning of your body if no return is found inside the body


 


please note the you also could have written your own anonymous function like below:

v('vendorCommission$USD',function(){return tb.vars.currentPrice$USD*0.1}).view();
vendorCommission = 123.4 USD





as we have seen above the function is created with row, col parameters.


This is of use if the function is used inside a table.


Please note the explicit usage of .style in order to set another format for the numbers
